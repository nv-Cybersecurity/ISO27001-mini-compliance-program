# ISO 27001 Mini Compliance Program – SecureHealth SaaS

## Overview
This project simulates building a compliance program for a healthcare SaaS company handling sensitive patient data (PHI).

## Framework
ISO 27001

## Scope
- AWS Cloud Infrastructure
- Web Application & APIs
- Patient Data Storage
- Employee Endpoints

## Deliverables
- Control Matrix (15+ controls)
- Risk Register
- Compliance Overview Document
- Access Control Policy
- Internal Audit Report

## Key Skills
- GRC (Governance, Risk, Compliance)
- ISO 27001 Controls
- Risk Assessment
- Gap Analysis
- Policy Development

## Summary
This project identifies security gaps and provides remediation strategies to improve compliance readiness and security posture.

## Sample Findings

- Identified lack of incident response plan (High Risk)
- Detected absence of centralized logging (Critical Risk)
- Found missing asset inventory and access review processes

## Tools & Frameworks
- ISO 27001 Annex A Controls
- Risk Assessment Methodology (Impact vs Likelihood)

## Project Overview
This mini compliance program demonstrates ISO 27001 implementation for a fictional SaaS company, SecureHealth. It includes a control matrix, risk register, sample policies, and a system architecture diagram.

## Sample Findings
- Incident Response Plan missing (High Risk)
- Centralized Logging not implemented (Critical Risk)
- Asset Inventory & Access Reviews incomplete (Medium Risk)

## Business Impact
Reduces risk of PHI breaches, improves audit readiness, and strengthens internal controls for the SecureHealth SaaS environment.
